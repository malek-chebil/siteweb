   ```bash

   ```

3. **Install essential packages**:
   ```bash
   apt install -y curl wget git vim ufw fail2ban unattended-upgrades
   ```

4. **Reboot system** (if kernel was updated):
   ```bash
   reboot
   # Wait for reboot, then reconnect via SSH
   ```

#### Step 4.1.3: Create Non-Root User

1. **Create new user**:
   ```bash
   adduser yourusername
   # Enter password (strong password)
   # Enter full name: (leave empty or use pseudonym)
   # Enter room number: (leave empty)
   # Enter work phone: (leave empty)
   # Enter home phone: (leave empty)
   # Enter other: (leave empty)
   # Is the information correct? [Y/n]: Y
   ```

2. **Add user to sudo group**:
   ```bash
   usermod -aG sudo yourusername
   ```

3. **Copy SSH keys to new user**:
   ```bash
   mkdir -p /home/yourusername/.ssh
   cp ~/.ssh/authorized_keys /home/yourusername/.ssh/
   chown -R yourusername:yourusername /home/yourusername/.ssh
   chmod 700 /home/yourusername/.ssh
   chmod 600 /home/yourusername/.ssh/authorized_keys
   ```

4. **Test new user**:
   ```bash
   su - yourusername
   # Verify you can use sudo:
   sudo whoami
   # Should output: root
   ```

#### Step 4.1.4: Configure SSH Security

1. **Backup SSH config**:
   ```bash
   sudo cp /etc/ssh/sshd_config /etc/ssh/sshd_config.backup
   ```

2. **Edit SSH config**:
   ```bash
   sudo nano /etc/ssh/sshd_config
   ```

3. **Configure SSH settings**:
   ```bash
   # Change SSH port (optional, but recommended)
   Port 2222  # Change from 22 to 2222 (or any port you want)

   # Disable root login
   PermitRootLogin no

   # Disable password authentication (use SSH keys only)
   PasswordAuthentication no
   PubkeyAuthentication yes

   # Disable empty passwords
   PermitEmptyPasswords no

   # Disable X11 forwarding
   X11Forwarding no

   # Limit login attempts
   MaxAuthTries 3

   # Set idle timeout
   ClientAliveInterval 300
   ClientAliveCountMax 2

   # Save and exit: Ctrl+X, Y, Enter
   ```

4. **Restart SSH service**:
   ```bash
   sudo systemctl restart sshd
   ```

5. **Test SSH connection** (from new terminal, don't close current session):
   ```bash
   ssh -i ~/.ssh/1984_hosting_key -p 2222 yourusername@YOUR_VPS_IP
   # Verify you can connect with new user and new port
   ```

6. **Update firewall** (if you changed SSH port):
   ```bash
   sudo ufw allow 2222/tcp
   sudo ufw delete allow 22/tcp  # Remove old port
   ```

**Important Notes**:
- âœ… Always test SSH connection before closing current session
- âœ… Keep backup SSH session open while testing
- âœ… Don't lock yourself out of server
- âœ… Use SSH keys (more secure than passwords)

### 5.1 Install PostgreSQL

#### Step 5.1.1: Install PostgreSQL

1. **Update package list**:

2. **Install PostgreSQL**:
   ```bash

   ```

3. **Check PostgreSQL status**:
   ```bash

   ```

4. **Enable PostgreSQL** (start on boot):
   ```bash
   sudo systemctl enable postgresql
   sudo systemctl start postgresql
   ```

#### Step 5.1.2: Configure PostgreSQL

1. **Switch to postgres user**:
   ```bash
   sudo -i -u postgres
   ```

2. **Create database and user**:
   ```bash
   # Create database user
   createuser --interactive --pwprompt yourdbuser
   # Enter password (strong password, save it securely)
   # Shall the new role be a superuser? [y/N]: N
   # Shall the new role be allowed to create databases? [y/N]: Y
   # Shall the new role be allowed to create more new roles? [y/N]: N

   # Create database
   createdb -O yourdbuser yourdbname

   # Exit postgres user
   exit
   ```

3. **Configure PostgreSQL for remote access** (if needed):
   ```bash
   sudo nano /etc/postgresql/14/main/postgresql.conf
   # Find: #listen_addresses = 'localhost'
   # Change to: listen_addresses = 'localhost'  # Only localhost for security
   # Save and exit
   ```

4. **Configure PostgreSQL authentication**:
   ```bash
   sudo nano /etc/postgresql/14/main/pg_hba.conf
   # Add at the end:
   # TYPE  DATABASE        USER            ADDRESS                 METHOD
   local   all             all                                     peer
   host    all             all             127.0.0.1/32            md5
   host    all             all             ::1/128                 md5
   # Save and exit
   ```

5. **Restart PostgreSQL**:
   ```bash
   sudo systemctl restart postgresql
   ```

#### Step 5.1.3: Configure PostgreSQL Security

1. **Set up SSL/TLS** (for encrypted connections):
   ```bash
   # Generate SSL certificate (self-signed)
   sudo openssl req -new -x509 -days 365 -nodes -text -out /etc/postgresql/14/main/server.crt -keyout /etc/postgresql/14/main/server.key -subj "/CN=yourdomain.com"

   # Set permissions
   sudo chmod 600 /etc/postgresql/14/main/server.key
   sudo chown postgres:postgres /etc/postgresql/14/main/server.key
   sudo chown postgres:postgres /etc/postgresql/14/main/server.crt
   ```

2. **Configure PostgreSQL for SSL**:
   ```bash
   sudo nano /etc/postgresql/14/main/postgresql.conf
   # Find: #ssl = off
   # Change to: ssl = on
   # Find: #ssl_cert_file = 'server.crt'
   # Change to: ssl_cert_file = 'server.crt'
   # Find: #ssl_key_file = 'server.key'
   # Change to: ssl_key_file = 'server.key'
   # Save and exit
   ```

3. **Restart PostgreSQL**:
   ```bash
   sudo systemctl restart postgresql
   ```

4. **Test PostgreSQL connection**:
   ```bash
   psql -U yourdbuser -d yourdbname -h localhost
   # Enter password when prompted
   # Test connection:
   SELECT version();
   # Exit: \q
   ```

**Important Notes**:
- âœ… Use strong passwords for database users
- âœ… Don't use default postgres user for applications
- âœ… Enable SSL/TLS for encrypted connections
- âœ… Restrict database access to localhost only
- âœ… Regularly update PostgreSQL
- âœ… Backup database regularly

       add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload" always;
       add_header X-Frame-Options "DENY" always;
       add_header X-Content-Type-Options "nosniff" always;
       add_header X-XSS-Protection "1; mode=block" always;
       add_header Referrer-Policy "strict-origin-when-cross-origin" always;

       # Frontend
       location / {
           root /var/www/yourclassifieds/frontend/dist;
           try_files $uri $uri/ /index.html;
       }

       # Backend API
       location /api/ {
           proxy_pass http://localhost:8000;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
       }

       # Security: Disable access to hidden files
       location ~ /\. {
           deny all;
       }
   }
